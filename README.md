# YouTube Channel Data Collector & Analyzer

Инструментарий для сбора метаданных видео с YouTube-канала и анализа объема текстовых данных (транскриптов) для использования в LLM (Large Language Models).

## 📌 Описание

Проект предназначен для автоматизации сбора информации о видео с конкретного канала (в данном случае — Lex Fridman). Система поддерживает как полную синхронизацию данных, так и инкрементальное обновление, чтобы минимизировать расход квот YouTube API. Также в проект включен модуль для точного подсчета токенов, что критически важно при подготовке данных для GPT-4/Claude.

## 🚀 Быстрый старт

### 1. Подготовка окружения

```bash
# Создание виртуального окружения
python -m venv venv

# Активация (Windows)
.\venv\Scripts\activate
# Активация (macOS/Linux)
source venv/bin/activate

# Установка зависимостей
pip install google-api-python-client python-dotenv tiktoken
```

### 2. Настройка

1. Делайте копию `.env` из `.env.example` в корне проекта.
2. Добавьте в него ваш API ключ от Google Cloud Console
3. В файле `config.py` можно изменить `CHANNEL_HANDLE` (например, на `@another_channel`), если нужно собирать данные с другого канала.

## 🛠 Использование

### Сбор данных о видео

Все скрипты запускаются из корня проекта.

**Первичный сбор всех данных (Full Sync):**
_Затирает существующий JSON и скачивает всё заново. Используйте для обновления статистики или смены канала._

```bash
python tools/full_sync.py
```

**Обновление базы (Incremental Update):**
_Сравнивает локальный JSON с данными YouTube и докачивает только новые ролики. Экономит квоту API._

```bash
python tools/update_videos.py
```

### Анализ текста

**Подсчет токенов для LLM:**
_Считает количество токенов в файле, используя алгоритм `cl100k_base` (стандарт для GPT-4)._

```bash
python tools/calk_tokens.py
```

## ⚙️ Технические особенности

- **Оптимизация квот:** Для получения детальной информации о видео используется батчинг (запрос по 50 ID за один раз), что снижает количество запросов к API в 50 раз.
- **Эффективность поиска:** При обновлении данных используется структура `set` для идентификаторов видео, что обеспечивает сложность поиска $O(1)$.
- **Точность токенизации:** Вместо примерного подсчета слов используется библиотека `tiktoken`, что позволяет точно определить, влезет ли текст в контекстное окно модели.

## ⚠️ Важные замечания

- **Квоты YouTube API:** Бесплатный лимит составляет 10,000 единиц в день. Рекомендуется использовать `update_videos.py` для регулярного обновления.
- **Безопасность:** Никогда не фиксируйте файл `.env` в системе контроля версий.
